- Real `DONE`: 5 items.

The project is moving in the right direction: the harness now has real helper code for validation, deployment-bundle generation, local-dev-site detection, GUI error filtering, CI summaries, Compose examples, and self-hosted workflow examples. The main risk is status inflation: several TODO entries are implemented as local dry-runs or contracts, not as production deployment or end-to-end CI-verified capabilities.

The most important next step is to run and archive the full GitHub Actions matrix results, then update `docs/VALIDATION_REPORT.md` and this audit with real CI evidence before promoting any status to fully verified `DONE`.

## Main misunderstanding

`docs/TODO.md` describes a plan and current status labels, but writing a capability into TODO does not mean it has been executed. This audit checks actual execution based on code, workflow definitions, generated-report paths, and documentation. Items that are only described are marked `DOCUMENTED ONLY`; items implemented only as dry-runs or local helpers are not treated as production-ready.
